Recording Studio Help

Record pādas of the Amarakosha and export them for distribution to students.

Overview

The Recording Studio is a standalone web page designed for teachers and advanced students who want to create audio recordings of Amarakosha pādas. Recordings can be exported as ZIP files and imported into the Practice App by students as named Audio Sets.

The studio shows one pāda at a time — its Devanagari text, section label, and position within the Varga. You record, review, and export entirely within the browser; no account or server upload is required.

Important: Recordings are held in memory for the current browser session only. They are not saved to disk automatically. Export a ZIP before closing or refreshing the page, or your recordings will be lost.

Selecting Content

Two dropdown menus at the top of the page let you navigate the text:

  • Kāṇḍa — one of the three main chapters (Svargakāṇḍa, Bhūmikāṇḍa, Sāmānyakāṇḍa).
  • Varga — a topical group within the selected Kāṇḍa (e.g. "Gods", "Trees", "Animals"). Changing the Kāṇḍa resets the Varga selection.

After selecting a Varga, the studio loads its first pāda automatically.

Navigating Pādas

Three navigation methods are available:

  • ◀ Prev / Next ▶ buttons — move one pāda at a time through all sections.
  • Pāda dot strip — a row of small tiles showing all pādas in the current section. Click any tile to jump directly to that pāda.

Recording

The large red circle button (⏺) controls recording.

  1. Navigate to the pāda you want to record.
  2. Tap the red button. Your browser will request microphone permission the first time — grant it.
  3. The button turns solid red (⏹) and a timer counts up while recording. Speak the pāda clearly.
  4. Tap the button again to stop. The recording is processed and saved immediately (converted to WAV for precise timing).

Playback & Review

After recording, the play button (▶) becomes active. Tap it to hear the recording. Tap again (while playing) to pause.

Use playback to verify the quality and completeness of each recording before moving on. If unsatisfied, simply record again — the new recording replaces the old one.

Deleting a Recording

Tap the × button (×) to delete the recording for the current pāda. The status returns to "Not recorded" and the pāda's dot returns to grey. This cannot be undone.

Progress Tracking

A progress bar and counter below the dot strip show how many pādas in the current Varga have been recorded:

  • Varga progress label with a N / M recorded count.
  • A green fill bar showing the percentage of Varga pādas covered.
  • The Download buttons show counts of recorded pādas for the current section, varga, and total.

Downloading Recordings

The Export section (tap the "Export ···" toggle on mobile) offers four download buttons:

⬇ Section 4
⬇ Varga ZIP 12
⬇ All recorded 18

This Pāda

Downloads the current pāda's recording as a single .wav file named after the GRETIL ID (e.g. 1.1.5.wav). Only active if the current pāda has been recorded.

Section ZIP

Downloads a ZIP containing WAV files for all recorded pādas in the current section. The number badge shows how many pādas in the section have recordings. Disabled if no pādas in the section have been recorded.

Varga ZIP

Downloads a ZIP containing WAV files for all recorded pādas in the current Varga (across all sections). This is the format expected by the Practice App's import feature. The number badge shows how many Varga pādas have recordings.

All Recorded ZIP

Downloads a ZIP containing every recording made in the current session, regardless of which Varga they belong to. Useful for a session spanning multiple Vargas. The number badge shows the total recording count.

All ZIP files use the STORE (uncompressed) format so that the Practice App can parse them without a decompression library. File sizes are slightly larger than compressed ZIPs but open reliably across all platforms.

Sharing with Students

The typical workflow for distributing recordings:

  1. Record pādas for one or more Vargas in the Recording Studio.
  2. Download the Varga ZIP (or All recorded ZIP for multi-varga sessions). Name the file something recognisable, e.g. amara_varga1_sharma.zip.
  3. Share the ZIP with students via email, WhatsApp, or any file-sharing service.
  4. Students open the Practice App, navigate to any pāda, and tap + Import in the Audio card.
  5. Students select the ZIP, give the set a name (e.g. "Pandit Sharma"), and tap Import. The recordings are available immediately as a selectable Audio Set.
Tip: WAV files inside the ZIP must be named with the GRETIL ID (e.g. 1.1.5.wav). Files exported by the Recording Studio use this naming automatically. Hand-prepared ZIPs must follow the same convention.

Mobile Tips

On small screens the layout adapts for easier recording:

  • The Kāṇḍa and Varga selectors stack vertically and take full width.
  • The record button is larger (64 px) and centred for comfortable one-thumb tapping.
  • The Play and Delete buttons appear below the record button as a secondary row.
  • The single-pāda download button (⬇) is hidden; use the Export ··· toggle instead to access all download options.
  • The pāda dot strip becomes a horizontal scrolling tab bar — swipe left/right to see all pādas in the current section.

Tips & Limits

  • Recording quality: Record in a quiet environment. The studio captures mono audio and converts to 16-bit PCM WAV at the device's native sample rate (typically 44.1 kHz or 48 kHz).
  • Session-only storage: Recordings live only in the current browser tab. Do not refresh or navigate away without exporting first.
  • 30-second limit per pāda: Each recording is capped at 30 seconds. Longer pādas should be re-read more quickly, or recorded in separate sessions and merged offline.
  • Microphone permission: The browser asks for permission on the first recording attempt. If you accidentally deny it, reset permissions in your browser settings (usually via the 🔒 icon in the address bar).
  • Browser compatibility: Works in any modern browser supporting the MediaRecorder API — Chrome, Edge, Firefox, and Safari 14.1+. Safari may use MP4 instead of WebM; both are converted to WAV transparently.
  • File naming: Downloaded WAV files are named by GRETIL ID (e.g. 1.1.5.wav). ZIP files are named by Varga ID and optional section sequence (e.g. amara_1-1.zip).
